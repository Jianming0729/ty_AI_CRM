# 🧹 周期性清理与代码瘦身协议 (Cleanup Protocol)

## 1. 治理目标
保持项目工程的“低熵值”，剔除在迭代过程中产生的中间产物、死逻辑以及临时调试脚本。

## 2. 清理范围 (RECURRING)
- **临时脚本**: `wecom-bridge/src/` 下所有的 `test_*.js`, `debug_*.js`, `local_test.js` 在功能合入主线并验证成功后，应移入 `test/` 目录或直接删除。
- **孤儿路由**: 停止使用的 WeCom 回调路由或 Intent 处理函数。
- **环境碎片**: 根目录下的 `.tar.gz`, `.sql` 导出文件，过期 `.env.temp`。
- **逻辑债务**: 带有 `// TODO: cleanup` 标记且已被新逻辑取代的代码块。

## 3. 执行频率
- **每次同步 (Sync Mode)**: 检查并清理当前功能路径下的调试残留。
- **每周例行**: 执行全域依赖扫描，剔除 `package.json` 中未引用的包。

## 4. SSOT 关联
清理动作必须记录在 `ACTIVE_INDEX.md` 的演进志中，防止误删。
